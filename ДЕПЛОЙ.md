# ФОРТОЧКА — деплой в облако (бесплатно)

Цель: сайт работает в интернете, открывается с любого устройства.
Связка: GitHub (код) + Vercel (хостинг сайта) + облачный Supabase (база) +
GitHub Actions (воркер по расписанию). Всё на бесплатных тарифах.

Все команды выполняются У ТЕБЯ в терминале (в папке app-taks), кроме шагов
в веб-интерфейсах GitHub/Vercel/Supabase.

================================================================
ЭТАП 0. Перед началом — пересоздать «засветившиеся» ключи
================================================================
В переписке светились ключи Groq, Gemini и облачного Supabase. Перевыпусти их,
старые отзови (по 1 минуте в кабинетах):
- Groq:   console.groq.com → API Keys → удалить старый, Create new
- Gemini: aistudio.google.com/apikey → удалить старый, Create new
- Supabase service_role: НЕ пересоздаётся, но он и так секретный — просто
  никуда его не вставляй открыто.

================================================================
ЭТАП 1. Перенести данные из локального Docker в облако (дамп)
================================================================
Локальный Supabase должен быть запущен (supabase start, Docker включён).

1.1. Выгрузить данные локальной базы в файл:
    supabase db dump --local --data-only -f backup-data.sql

    (если хочешь и схему тоже — отдельно:
     supabase db dump --local -f backup-schema.sql)

1.2. Применить схему к облачной базе (если ещё не applied):
    Открой облачный проект (vecmunfotagjmfhxreob) → SQL Editor →
    вставь содержимое supabase/schema.sql → Run.

1.3. Залить данные в облако. Нужна строка подключения облака:
    Supabase → Project Settings → Database → Connection string (URI).
    Затем:
    supabase db dump  # убедиться что backup-data.sql на месте
    psql "ПОДКЛЮЧЕНИЕ_К_ОБЛАКУ" -f backup-data.sql

    Если psql не установлен — можно через Supabase Studio: SQL Editor →
    вставить содержимое backup-data.sql → Run (для небольшого объёма ок).

    ВНИМАНИЕ: при конфликтах id/foreign keys — сказать Claude, разрулим.

================================================================
ЭТАП 2. Залить код на GitHub
================================================================
2.1. Создать репозиторий на github.com (кнопка New, имя fortochka, Private).
     НЕ добавляй README/gitignore при создании (они уже есть локально).

2.2. В терминале (в app-taks):
    git init
    git add .
    git commit -m "ФОРТОЧКА: первый коммит"
    git branch -M main
    git remote add origin https://github.com/ТВОЙ_ЛОГИН/fortochka.git
    git push -u origin main

    .env.local НЕ попадёт в GitHub (он в .gitignore) — это правильно.

================================================================
ЭТАП 3. Развернуть сайт на Vercel
================================================================
3.1. vercel.com → Sign up через GitHub.
3.2. Add New → Project → выбрать репозиторий fortochka → Import.
3.3. Framework Next.js определится сам. Перед Deploy открой
     Environment Variables и добавь (значения — из облака и кабинетов ИИ):
        NEXT_PUBLIC_SUPABASE_URL      = https://vecmunfotagjmfhxreob.supabase.co
        NEXT_PUBLIC_SUPABASE_ANON_KEY = (облачный anon, формат eyJ...)
        SUPABASE_SERVICE_ROLE_KEY     = (облачный service_role, eyJ...)
        GROQ_API_KEY                  = (новый)
        GEMINI_API_KEY                = (новый)
        ADMIN_KEY                     = (длинная строка)
        SITE_URL                      = https://fortochka.vercel.app (адрес дадут после деплоя)
        # Telegram — позже, когда настроишь
3.4. Deploy. Через пару минут получишь адрес вида fortochka.vercel.app —
     сайт доступен всем.
3.5. ВАЖНО: в next.config.mjs поменять `unoptimized: true` → `false`
     (включит оптимизацию картинок на Vercel). Закоммить и запушить —
     Vercel передеплоит сам.

================================================================
ЭТАП 4. Воркер по расписанию (GitHub Actions)
================================================================
Чтобы новости собирались сами, без твоего ПК. Создать файл
.github/workflows/worker.yml (Claude поможет составить). Он будет по cron
(например каждые 2 часа) запускать collect→filter→draft, используя те же
секреты (их добавить в GitHub: Settings → Secrets and variables → Actions).

================================================================
ВАЖНЫЕ ЗАМЕТКИ
================================================================
- Облачный Supabase из РФ для УПРАВЛЕНИЯ может требовать VPN, но сайт на
  Vercel (за границей) достучится до базы без проблем.
- Бесплатный Supabase засыпает после ~недели без активности; регулярный
  воркер его будит.
- Локальная разработка остаётся рабочей: supabase start + npm run dev.
  Облако и локалка не мешают друг другу (разные .env).
- Если на любом шаге ошибка — скопируй текст (без ключей) и покажи Claude.

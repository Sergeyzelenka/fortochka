# ФОРТОЧКА · прогон воркера на живых данных

Пошаговая памятка для первого реального запуска пайплайна
(сбор → ИИ-фильтр → ИИ-редактор). Без Telegram — публикация в канал
пока пропускается, статьи лягут в очередь админки.

## 1. Ключи (всё бесплатно, без карты)

- **Groq** (ИИ-фильтр): console.groq.com → API Keys → Create → ключ `gsk_...`
- **Gemini** (ИИ-редактор): aistudio.google.com → Get API key → `AIza...`
- **Supabase** (база): supabase.com → New project →
  Project Settings → API (взять URL, anon, service_role) →
  SQL Editor → выполнить содержимое `supabase/schema.sql`

## 2. Файл `.env.local` (создать в папке app-taks)

```
NEXT_PUBLIC_SUPABASE_URL=https://твой.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=твой_anon
SUPABASE_SERVICE_ROLE_KEY=твой_service_role
GROQ_API_KEY=gsk_...
GEMINI_API_KEY=AIza...
ADMIN_KEY=придумай_длинную_строку
SITE_URL=http://localhost:3000
# Telegram пока пусто — публикация в канал пропустится:
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHANNEL_ID=
TELEGRAM_ADMIN_CHAT_ID=
```

⚠️ Ключи — это секреты. Не отправляй их в чат, держи только в этом файле.
Файл `.env.local` не попадает в git (он в .gitignore).

## 3. Запуск (в терминале, в папке app-taks)

```bash
npm install                # один раз

npm run worker:collect     # RSS → статьи "found"
npm run worker:filter      # Groq: оценка и отсев (found → filtered / rejected_by_ai)
npm run worker:draft       # Gemini: переписывает по-русски (filtered → pending_review)
```

Или всё разом: `npm run worker`

## 4. Посмотреть результат

```bash
npm run dev
```

Открыть в браузере:
- Лента: http://localhost:3000
- Админка (очередь на апрув): http://localhost:3000/admin?key=ТВОЙ_ADMIN_KEY

В админке — статьи в статусе pending_review со скором ИИ, превью текста,
кнопки «Опубликовать» / «Отклонить».

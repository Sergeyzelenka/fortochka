## 1. Что это за проект

**ФОРТОЧКА** — позитивное новостное медиа («только хорошие новости») для
русскоязычной аудитории. Полностью автоматизированный конвейер:

```
RSS-источники → ИИ-фильтр (отсев) → ИИ-редактор (переписывание по-русски)
            → ручной апрув главреда в админке → публикация на сайт + Telegram
```

Бренд раньше назывался ТАКС, переименован в ФОРТОЧКА (слоган «глоток свежих
новостей»). Причина: ТАКС созвучно с ТАСС — риск для новостного бренда.

Расположение: `D:\Боты\ТАКС\ТАКС\app-taks` (Next.js-проект).
Папка `D:\Боты\ТАКС\ТАКС\` — корень, там лежат демо-файлы полянки (см. ниже).

## 2. Стек и архитектура

- **Сайт + админка**: Next.js 14 (App Router) + React + TypeScript. Деплой задуман на Vercel.
- **База**: Supabase (Postgres). СЕЙЧАС используется ЛОКАЛЬНЫЙ Supabase через
  Docker (`supabase start`), потому что у пользователя в РФ были проблемы с
  облаком. Облачный проект тоже создан (`vecmunfotagjmfhxreob.supabase.co`),
  но для тестов работаем локально.
- **Воркер**: Node/TypeScript, запускается через `tsx`. Шаги: collect → filter → draft.
- **ИИ**: Groq (Llama 3.3 70B) — фильтр; Google Gemini 2.5 Flash — редактор. Ключи бесплатные.
- **Telegram**: публикация в канал через Bot API (опционально, сейчас не настроен до конца).

### Структура файлов
- `app/page.tsx` — лента новостей (главная, все рубрики)
- `app/cat/[slug]/page.tsx` — лента рубрики, своя og-метадата, пререндерится статически
- `app/article/[slug]/page.tsx` — страница статьи + generateMetadata (og:image, og:type=article)
- `app/admin/page.tsx` + `components/AdminQueue.tsx` — админка: очередь на апрув
- `app/admin/sources/page.tsx` + `components/SourcesAdmin.tsx` — управление RSS-источниками
- `app/admin/logs/page.tsx` — экран pipeline_log с фильтрами по шагу/статусу
- `app/api/admin/decide/route.ts` — API: опубликовать/отклонить + revalidatePath
- `app/api/admin/sources/route.ts` — GET/POST/PATCH/DELETE для источников
- `app/sitemap.ts` + `app/robots.ts` — SEO
- `app/layout.tsx` — глобальная og/twitter метадата + metadataBase из SITE_URL
- `components/NewsFeed.tsx` — общий рендер ленты (lead + briefs + grid), используется главной и /cat

- `worker/steps/{collect,filter,draft}.ts` — шаги пайплайна

## 5. Как запустить локально

Нужен запущенный **Docker Desktop**. Терминал в папке `app-taks`.

```bash
supabase start              # поднять локальную БД (данные сохраняются между запусками)
npm run dev                 # сайт на http://localhost:3000 (занимает терминал!)
```

Воркер (во ВТОРОМ терминале, пока сайт крутится в первом):
```bash
npm run worker:collect      # RSS → found
npm run worker:filter       # Groq отбирает (found → filtered/rejected_by_ai)
npm run worker:draft        # Gemini переписывает (filtered → pending_review)
npm run worker:resanitize   # одноразово пройти sanitize по всем старым body_html
```
filter/draft берут новости пачками — для обработки всего запусти 2-3 раза.

Админка (ключ = ADMIN_KEY из .env.local):
- `http://localhost:3000/admin?key=fortochka2026secret` — очередь на апрув
- `http://localhost:3000/admin/sources?key=...` — RSS-источники
- `http://localhost:3000/admin/logs?key=...` — логи воркера

SQL к локальной базе: `supabase db query "..."` (ОДИН запрос за раз — несколько
команд CLI не принимает).

## 6. Важные нюансы и грабли

- **Обрывы записи файлов.** В ходе разработки большие файлы (engine.ts, sound.ts,
  globals.css, миграция) НЕОДНОКРАТНО обрезались/портились при записи (потеря
  хвоста, null-байты, битая кириллица). ВСЕГДА после правки большого файла
  проверяй: баланс скобок, что файл заканчивается корректно, нет null-байтов,
  компилируется ли (`npx tsc` или esbuild). При сборке демо-HTML проверяй
  `node --check` на вырезанном JS.
- **Кириллица в SQL.** Длинное тире «—» и спецсимволы в комментариях ломали
  миграцию. Комментарии в SQL держи на латинице или без экзотики.
- **Supabase: новые vs легаси ключи.** Новые ключи (`sb_publishable_`/`sb_secret_`)
  не работают со старой версией @supabase/supabase-js. Используй легаси-ключи
  формата `eyJ...` (для локалки — стандартные demo-ключи CLI).
- **Сеть в РФ.** Часть зарубежных RSS и Docker-реестров (ecr.aws, jsr.io)
  недоступны без VPN — отсюда отключённые сервисы Supabase и Phys.org 451.
- **Безопасность.** Облачные ключи (от supabase.co) и ключи Groq/Gemini
  засветились в переписке ранее — пользователю стоит их пересоздать перед
  публичным запуском. Локальные demo-ключи (127.0.0.1) безопасны.
- **Два терминала.** `npm run dev` занимает терминал — воркер и SQL запускай во втором.

### Как устроен сбор (важно для понимания)
- **collect (сбор)** — БЕЗ ИИ, квоты НЕ тратит. Просто читает RSS-ленты источников
  (rss-parser), отсеивает дубли, кладёт в базу со статусом found.
- **filter** — Groq читает found-статьи, решает позитив/нет и СРАЗУ присваивает категорию.
  Хорошие → filtered, плохие → rejected_by_ai. Тратит квоту Groq.
- **draft** — Groq или Gemini переписывают filtered по-русски → pending_review. Тратит квоту.
- **апрув** — вручную, публикация на сайт + Telegram.

### Выбор источников для сбора RSS (Вариант Б — галочки по источникам)
Пользователь хочет управлять сбором: вручную выбирать, ИЗ КАКИХ конкретных
источников собирать новости (чтобы разбавлять перекос космос/медицина и тянуть
больше из доброты/лайфхаков/технологий).
ПОЧЕМУ так, а не «фильтр по категории»: на стадии сбора у статьи ещё НЕТ категории
(её ставит ИИ позже на фильтре). Поэтому отбор делаем на уровне ИСТОЧНИКОВ.

ЧТО СДЕЛАТЬ (Вариант Б, выбран пользователем):
- В /admin/sources у каждого источника добавить переключатель «участвует в сборе»
  (отдельно от enabled, либо переиспользовать enabled — обсудить). Скорее всего
  завести новое булево поле sources.collect_enabled (default true) ИЛИ просто
  трактовать существующее sources.enabled как «участвует в сборе».
- collect.ts читает только источники с этим флагом (сейчас, вероятно, берёт enabled).
- В админке (SourcesAdmin.tsx) — чекбоксы массового вкл/выкл, чтобы быстро
  отметить «собирать только из этих».
ПРОВЕРИТЬ: как collect.ts сейчас выбирает источники (enabled?), есть ли уже
поле под это. Не плодить лишних полей если enabled уже подходит.
